## 🟢 5월말~6월 (W22~W25, 2026-05-26~06-22)

### 6. 백로그 035 — 공통 표준 포맷 도입 ★ 가장 큰 root cause 해결

**Phase 1: 명세 확정** (1~2일)
- 엑셀 명세 완성 (형식 작성 중)
- dataclass 설계: `StandardProject` (7 필드 + 메타)
- 7개 source의 `site_type` 분류 (delegated_hub / self_hosted)
- 페이지 매칭 전략 명세 (목록↔상세, 지원사업명 기준)

**Phase 2: DB 마이그레이션** (1일)
- ALTER TABLE biz_projects ADD COLUMN apply_end, native_id, source_seq, business_start, business_end
- backfill 스크립트 + 검증

**Phase 3: 7개 커넥터 어댑터 구현** (3~5일)
- 각 커넥터에 `to_standard_project()` 함수
- delegated_hub: raw에서 organization 추출
- self_hosted: DEFAULT_ORG 하드코딩 (jbtp='전북테크노파크' 등)
- 페이지 매칭 로직 구현 (목록 페이지에서 연번 + 제목 수집 → 상세 페이지 제목으로 역매칭)

**Phase 4: 검증** (1일)
- 모든 row의 7필드 NOT NULL 비율 100% 확인
- 위젯 표시 정합성 검증 (spSeq= 깨짐 0건, 마감 공고 노출 0건)

**Phase 5: 위젯·메일 전환** (2일)
- raw 직접 접근 제거, 표준 포맷만 사용
- Apps Script 메일 발송도 표준 포맷 기반

**자연 흡수되는 백로그**:
- 백로그 031 dedup → native_id 자동 적용
- 백로그 034 jbexport notice_no → 동일 패턴
- 마감 필터 누락 → status 일관 채움
- 카테고리 분류 일관성
