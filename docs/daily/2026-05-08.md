## 백로그 036 — bizplnner Task wrapper proxy 자동 시작

### 진단

`auto_run.bat` (5/1 마지막 수정, 116 bytes) 단순 wrapper — proxy 안 띄움:
```bat
@echo off
cd /d "C:\Users\custo\OneDrive\바탕 화면\커서앱통합_v1"
py run_all.py >> logs\auto_run.log 2>&1
```

bizplnner Task 의 Action 도 `python.exe run_all.py` 직접 호출 — auto_run.bat 거치지 않음.

→ proxy 자동 시작 메커니즘이 어디에도 없음 (run_all.py / auto_run.bat / 시작 프로그램 / 별도 Task 모두 X).

### 새 wrapper 설계 (4단계)

1. **proxy 살아있나 체크** (`curl -s -o NUL -w "%{http_code}"`, 200/405 인정)
2. **죽어있으면 자동 시작** (`start "jbexport_proxy" /min py ...`, max 30s 응답 대기)
3. **run_all.py 실행** (기존 동작 그대로)
4. **wrapper 가 띄운 proxy 만 정리** (사용자 수동 proxy 보존)
   - `taskkill /F /FI "WINDOWTITLE eq jbexport_proxy*"`
   - `netstat fallback`: port 5001 LISTENING PID kill

### 인코딩 fix (재작성)

초안 작성 시 Write tool 이 UTF-8 저장 → cmd.exe `'rt' is not recognized` 깨짐. 재작성 정책:

| 항목 | 정책 |
|---|---|
| 인코딩 | ASCII strict (`encode("ascii")`) |
| BOM | 없음 (첫 3 byte = `@ec`) |
| line ending | CRLF |
| 한글 경로 | hardcoding 회피 — `cd /d "%~dp0"` |
| chcp 명령 | 안 씀 |
| echo 메시지 | 영어 only |

`cmd /c type auto_run.bat` 깨짐 0건 / non-ASCII bytes 0 / size 2459 bytes.

### Task Action 변경 (관리자 권한 필요)

- 일반 PowerShell `Set-ScheduledTask` → ❌ Access denied
- 일반 PowerShell `schtasks /change` → ❌ 동일
- **GUI taskschd.msc** → ✅ 사용자 직접 변경 완료
  - bizplnner → 동작 → 편집:
    - 프로그램/스크립트: `cmd.exe`
    - 인수: `/c auto_run.bat`
    - 시작 위치: `C:\Users\custo\OneDrive\바탕 화면\커서앱통합_v1`

### 검증

`schtasks /run /tn "\bizplnner"` 즉시 trigger → 정상 실행 확인.
