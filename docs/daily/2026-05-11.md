### v1 schema 변경 — ALTER TABLE 2건

```sql

ALTER TABLE biz_projects ADD COLUMN synced_at TIMESTAMP;
```

`synced_at` 은 마지막 성공 sync 시각 (디버깅 / 재시도 추적 / 통계 재료).

### INSERT/UPDATE 경로별 동작 요약

| 경로 | synced_to_render | synced_at |
|---|---|---|
| connector INSERT (신규) | 0 (default) | NULL |
| appy UPDATE (ai_summary 백필 등) | **0 으로 reset** | NULL reset |
| sync_to_render.py 성공 | 1 | CURRENT_TIMESTAMP |
| sync 실패 | (미갱신) | (미갱신) |

→ flag reset 로직이 핵심. INSERT 는 default 로 충족, UPDATE 경로는 명시적 reset 코드 필요 (Phase 2.1a 작업 시 검토).
