## 3. 가장 큰 위험 발견

- **네트워크 실패 823건 = 57.4%** (1,433 대상 중)
- 실패 유형 혼재:
  - `ConnectionResetError(10054)` — 원격 호스트 강제 연결 종료
  - `NameResolutionError` — DNS 해석 실패 (`getaddrinfo failed`)
  - `SSLV3_ALERT_HANDSHAKE_FAILURE` — SSL 핸드셰이크 실패
- **DNS 해석 실패가 결정적 단서** — 서버 rate limit이라면 보통 연결을 끊지 DNS는 살아 있음. DNS가 안 되는 건 클라이언트/네트워크 측 불안정 신호 → **박람회장 네트워크 불안정** 의심.
- 시뮬 **위험 B**("rate limit, 확률 낮음")가 rate limit + 네트워크 불안정 복합으로 현실화.
