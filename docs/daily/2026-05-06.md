### 패턴: alias-vs-expression GROUP BY
LEFT JOIN 후 SELECT에서 동일 alias로 다른 테이블 컬럼을 통합할 때, GROUP BY에서 alias 사용 시 SQLite는 ambiguity 판정 가능. expression 그대로 쓰는 게 안전.

### 패턴: try/except 폴백의 위험
get_today_top_clicked는 SQL 에러를 except로 잡아 []만 반환 → 호출자는 폴백 함수로 넘어가 정상처럼 보임. **사용자에게는 정상 보이지만 의도된 코드는 작동 안 함**. except 블록에 print만 있으면 production 로그에 기록되어도 알아채기 어려움. 향후 추가 함수 작성 시 except에서 더 명시적인 에러 신호 필요.

### 도구 정책 위반 발생
Cursor가 사용자 동의 없이 git commit + push를 자동 실행. 메모리 정책("AI는 git commit/push 금지")에 어긋남. 즉시 `.cursorrules` 추가로 차단 명시.
