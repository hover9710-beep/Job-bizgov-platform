# 데이터 출처 신뢰성 다층 노출 전략

**작성**: 2026-05-17 EOD 사후 발견
**상태**: 백로그 (5/18 작업 묶음 일부 + W21+ 일부)

---

## 배경

BizGovPlanner 의 데이터 출처:

- 공식 API 1개: 정부24 기업마당 (bizinfo)
- 자체 크롤러 7개:
  - K-Startup (창업진흥원)
  - JBTP (전북테크노파크)
  - JBBI (전북바이오융합산업진흥원)
  - jbexport (전북경제통상진흥원)
  - aT (한국농수산식품유통공사)
  - 한국수산회 (kseafood)
  - 전북외테크노파크 (JBTP 유관)

응모서/평가위원에게 어필 포인트:

- 공식 API 활용 = 정당성
- 자체 크롤링 7건 = 노동 가치 + 솔로 개발자 1년 누적
- 매일 06:00 자동 갱신 = 운영 신뢰성

## 다층 노출 전략

### 1차 — 메인 상단 신뢰성 배너 (필수)

- 위치: 위젯 위쪽, 카운트 카드 위
- 작업 시간: 20분
- 시스템 영향: 매우 낮음 (HTML + CSS)

디자인 권장 [B]:

```
┌─────────────────────────────────────────┐
│  📊 데이터 출처                          │
│  ✓ 정부24 기업마당 (공식 API)            │
│  ✓ 7개 기관 자체 수집 (매일 06:00 갱신)  │
└─────────────────────────────────────────┘
```

또는 상세 [C]:

```
┌─────────────────────────────────────────┐
│  ✅ 공식 API 1 + 자체 수집 7            │
│  기업마당(공식) + K-Startup, JBTP,       │
│  전북경제통상진흥원, aT,                  │
│  전북바이오, 한국수산회,                  │
│  전북외테크노파크                         │
└─────────────────────────────────────────┘
```

### 2차 — 상세 페이지 메타 정보

- 위치: 각 공고 상세 페이지의 메타 영역
- 작업 시간: 20분
- 시스템 영향: 매우 낮음 (template 조건문)

source 별 다른 표시:

- case 1: bizinfo
  - "데이터 출처: 정부24 기업마당 (공식 API)"
- case 2~8: 자체 크롤러
  - "데이터 출처: [기관명] 공식 사이트 (자체 수집, 매일 갱신)"

기관명 매핑 (정정 반영):

| source | 표시 명칭 |
|---|---|
| kstartup | K-Startup |
| jbtp | 전북테크노파크 (JBTP) |
| jbbi | 전북바이오융합산업진흥원 (JBBI) |
| jbexport | 전북경제통상진흥원 |
| at_global | 한국농수산식품유통공사 (aT) |
| kseafood | 한국수산회 |
| jbtp_related | 전북외테크노파크 |

### 3차 — /about 페이지 (응모서 후)

- 위치: 신규 페이지 /about
- 작업 시간: 1시간
- 시스템 영향: 중간 (신규 라우트)

내용:

- BizGovPlanner 비전
- 8개 기관 통합 워크플로우
- 솔로 + AI 협업 모델
- 데이터 출처 명세 (정정 반영)
- 누적 워크플로우 자산 (시뮬 누적 시스템)

진행 시점: W21+ (7/3 응모서 후)

## 시너지 (다른 차별점과)

이 전략 + 기존 차별점:

- "확인 필요" 정확성 우선 → 데이터 출처 명시로 강화
- 시뮬 누적 시스템 → 워크플로우 자산 어필
- AI 시대 솔로 워크플로우 → 1인의 노동 가치 어필
- 이재명 대통령 5/15 X 발언 → 시대적 정합성
- 데이터 출처 신뢰성 → 정당성 + 노동 가치

→ 5가지 차별점 통합 = 응모서 핵심 메시지
