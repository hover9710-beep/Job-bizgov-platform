## 핵심 차별점 (5/17 마라톤 누적 + 5/19 통합 갱신, 9가지)

1. **정확성 우선** — "확인 필요" 분류 정책, 사용자 가설 정정 6건 (5/17 마라톤 + 5/19 통합). 확인 필요 **≈100% = 기업마당 단일 문제** (PC DB 실측 99.95%, 2,125/2,126건). 다른 7개 자체 크롤러는 status 처리 완벽.
   - **Phase 3 본 구현 실측 (5/20~5/21) ⭐ NEW**: bizinfo 본문 `--enrich-detail` enrich + 무마감 공고 status 재분류 + enrich 야간 파이프라인 통합(영구화)까지 적용. **파싱 정확성 100%** (fetch 1,474건 전수 정상 추출, 무마감 공고는 정당하게 마감일 없음으로 분류). bizinfo "확인 필요" **2,302 → 905건 (−61%, 비율 29.8%)** — 야간 파이프라인에 통합되어 **매일 자동 유지**.
2. **누적 학습 워크플로우** — 시뮬 누적 시스템 (entry 8건), **사용자 가설 정정 누계 13건** (5/17 마라톤 6 + 통합 시뮬 1 + 5/19 사이트 캡처 3 + 5/19 PoC 사전 시뮬 1 + 5/20 코드 read 1 + 5/20 PoC 완료 1). 대표 사례:
   - 9번째 (5/19): bizinfo 본문이 첨부서류 역할 → Phase 3.0 작업량 4~6h → 1~2h
   - 10번째 (5/19, 사이트 캡처 KIAT/KICET): 기관별 본문 상이, "신청기간" 만 → Phase 3.0 정규식 4종
   - 11번째 (5/19 PoC 사전 시뮬): 정규식 4종 적용 전 DB 실측 → `description` 컬럼 = 날짜 문자열 (본문 X), 본문 자체가 DB 부재. **`connector_bizinfo.py` 에 `fetch_detail()` + `--enrich-detail` 이미 존재** → Phase 3.0 = 기존 스크립트 실행만 (코드 변경 0).
   - 12번째 (5/20 코드 read): `--enrich-detail` 명세 4건 정정 — DB 미접근·JSON-only, `--dry-run`/`--limit` 플래그 부재 등. 잘못된 명령 직진 차단.
   - **13번째 (5/20 PoC 완료, NEW)**: 가설 "enrich 실행 = Phase 3.0 완성". 실측 — enrich 결과가 야간 크롤·`update_db`에 의해 **매일 wipe**됨(비영구). → **Phase 3 본 구현의 핵심 = "영구화"**. PoC 실행 ≠ 본 구현임을 30분 검증으로 명확화.
   → **'선 분석, 후 진행' 본능의 진짜 작동**. "단순 시스템 구축 ≠ 누적 학습 워크플로우"
3. **AI 시대 솔로 워크플로우** — Claude/GPT-4o-mini 협업, 1인 → 전국 서비스 가능 증명
4. **시대적 정합성** — 이재명 5/15 X "창업중심 국가", 정부 5대 키워드 정확 매칭
5. **데이터 출처 신뢰성 + 자체 크롤러 운영** — 공식 API 1 (정부24 기업마당) + 자체 크롤러 7 **정기 운영** (K-Startup, 전북테크노파크, 전북바이오융합원, 전북경제통상진흥원, aT수출통합지원시스템, 수산식품수출지원시스템, 전북외테크노파크), 매일 06:00 KST 자동 갱신, 자체 모니터링 (크롤러 fail 알림) — **단순 수집 X, 자동 운영 ✓**
6. **개발자 본인이 첫 사용자 + 운영 모니터링** — 운영자 = 사용자 = 검증자

8. **Phase 3 = 본문 AI 분석 시작점** ⭐ NEW — 메타 AI 100% → 본문 AI 진입, BizGovPlanner AI 의 진짜 시작점 (아래 상세)
9. **Phase 4 = 회사 매칭 추천** ⭐ NEW — 본문 텍스트 + target_company 매칭, 단순 추천 X 본문 기반 맞춤 추천 (아래 상세)
